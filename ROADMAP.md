# Roadmap: Обозреватель Технической Документации

## Видение

Построить платформу библиотеки технической документации с HedEx-подобной навигацией и современными сценариями с использованием искусственного интеллекта для команд EPC/5GC.

## Принципы

- Быстрый прототип: сначала Python + FastAPI + OpenSearch
- Масштабирование через метрики: оптимизировать только подтвержденные узкие места
- Искусственный интеллект должен быть привязан к источникам и поддаваться аудиту
- Пользовательский интерфейс должен оставаться структурированным и предсказуемым для инженерных пользователей

## Журнал изменений

### 2026-06-06 — Закрыт риск совместимости зависимостей на Python 3.14

- Причина: установка старого стека `pydantic-core` падала из-за несовместимости PyO3/jiter.
- Решение: обновлены версии FastAPI/Starlette/Pydantic и добавлен `httpx` для тест-клиента.
- Результат: `python -m pip install -r requirements.txt` и `python -m pytest -q` выполняются успешно.

### 2026-06-06 — Подтверждены критерии выхода Фазы 0 и Фазы 1

- Проверка Фазы 0: `python scripts/phase0_gate.py` -> `PHASE0_GATE=PASS`.
- Подготовка данных Фазы 1: `python scripts/phase1_seed_docs.py --count 300`.
- Проверка Фазы 1: `python scripts/phase1_gate.py --offline-search --min-docs 300 --search-runs 30 --p95-ms 300` -> `PHASE1_GATE=PASS`.

### 2026-06-06 — Старт Фазы 2: создан клиент Next.js + TypeScript

- Добавлено отдельное приложение `frontend` на Next.js (App Router) и TypeScript.
- Реализованы базовые клиентские сценарии: проверка `GET /health` и тестовый `POST /api/v1/search`.
- Добавлена конфигурация `NEXT_PUBLIC_API_BASE_URL` через `.env.local`.

### 2026-06-06 — Реализован пункт 2 Фазы 2 (документация и чтение)

- Добавлена левая древовидная навигация по `product -> release -> domain -> topic`.
- Добавлены breadcrumb и переключение версии релиза в клиенте.
- Реализована базовая страница чтения документа с якорями по разделам (chunks).
- Интеграция с текущими backend-маршрутами выполнена через Next.js proxy API:
	- `GET /api/documents/tree` -> `GET /api/v1/documents/tree`
	- `GET /api/documents/{docId}` -> `GET /api/v1/documents/{doc_id}`

### 2026-06-06 — Закрыты оставшиеся пункты Фазы 2

- Добавлены сохраненные поиски в дереве документации (сохранение в localStorage).
- Добавлены закрепленные разделы документа (якоря с фиксацией в localStorage).
- Подтверждена адаптивность макета для desktop/mobile (responsive layout sidebar/content/anchors).

### 2026-06-06 — Старт Фазы 3: retrieval-конвейер по чанкам

- `/api/v1/ai/ask` переведен с заглушки на retrieval по индексированным чанкам OpenSearch.
- Добавлена фильтрация контекста по `context_doc_ids` и лимит `max_citations`.
- Ответ теперь формируется на основе найденных источников с реальными citation/snippet.
- Добавлен smoke-тест `test_ai_context_retrieval_pipeline`.

### 2026-06-06 — Расширение Фазы 3: режимы и строгая source-policy

- Добавлены режимы шаблонов ответа: `explain`, `compare`, `diagnose` поверх retrieval.
- Включена строгая политика обязательных источников: без citation ответ блокируется.
- Добавлены трассировочные поля ответа: `trace_id`, `retrieval_stats`, `blocked`, `source_policy`.
- Логирование `/api/v1/ai/ask` расширено mode/trace/block статусом.

### 2026-06-06 — Завершение Фазы 3: цикл обратной связи AI

- Добавлен endpoint `POST /api/v1/ai/feedback` для оценки ответа (`like`/`dislike`) по `trace_id`.
- Реализовано хранение feedback в SQLite (`ai_feedback`) с индексом по `trace_id`.
- Добавлен интеграционный smoke-тест `test_ai_feedback_cycle_by_trace_id`.

### 2026-06-06 — Старт Фазы 4: легкая ролевая модель и аудит админ-действий

- Добавлена lightweight RBAC-модель с ролями `admin`/`user` в учетных записях и токенах входа.
- Защищены административные маршруты по Bearer-токену и проверке роли `admin`.
- Добавлен журнал `admin_audit_log` для создания, обновления и удаления пользователей.
- Добавлен endpoint просмотра журнала аудита для роли `admin`.

## Ключевые KPI

- Пропускная способность конвейера загрузки (документов/час)
- Задержка поиска (p50/p95/p99)
- Качество поиска на эталонных запросах (релевантность Top-5)
- Доля ответов искусственного интеллекта с валидными ссылками на источники
- Успешность пользовательских задач (быстро найти нужный раздел)
